# Rules - Luật Cứng Của MeowAI

Thư mục này chứa các luật cứng để giữ project không bị lệch hướng.

Dev hoặc Codex phải đọc thư mục này trước khi sửa code.

## Thứ Tự Đọc

1. `CORE_PROMPT.md` - vai trò, mục tiêu và cách làm việc.
2. `PHASE_GATES.md` - phase nào được làm, khi nào được qua phase mới.
3. `FILE_EDITING_RULES.md` - file nào được sửa, file nào cần cẩn thận.
4. `AI_TRAINING_RULES.md` - nguyên tắc train AI cho đồ án.

## Nguyên Tắc Chung

- Project phải dễ hiểu với sinh viên và giảng viên.
- Code ưu tiên rõ ràng hơn là quá tối ưu.
- Không đưa công nghệ quá phức tạp nếu chưa cần.
- Không sửa tài liệu hàng loạt chỉ để "cho đẹp".
- Mỗi phase phải có kết quả chạy được.
